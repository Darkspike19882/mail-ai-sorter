## Phase Scope: SRCH-01, SRCH-02, SRCH-03, SRCH-04

This phase enhances the existing search bar in `templates/inbox.html` and adds a new **AI-Q&A panel** accessible from the inbox page. The design builds entirely on the existing inbox three-pane layout and Alpine.js data model — no separate search page.

### Existing State (from codebase audit)

| What | Where | Current State |
|------|-------|---------------|
| Search input | `inbox.html` line 103 | Simple text input, debounced at 250ms, hits `/api/search?q=` |
| Search API | `inbox_routes.py` line 74 | FTS5 MATCH query against `emails_fts`, returns 20 results, supports `?category=` |
| Search results | Alpine.js `mails[]` array | Results replace inbox list; searchActive banner shown |
| Account/folder filter | Alpine.js `isUnified`, `currentAccount`, `currentFolder` | Selecting a folder clears search; no combined filter during search |
| RAG engine | `rag_engine.py` | Full pipeline: FTS5 → Ollama → grounded answer with sources |
| RAG API | `rag_routes.py` | `POST /api/rag/query` with `{query, limit}`, `GET /api/rag/status`, `POST /api/rag/reindex` |
| FTS5 schema | `index.py` lines 27–75 | Searches `from_addr`, `subject`, `keywords`, `snippet`; no account/folder columns in FTS |

## Interaction Design

### 2.1 Enhanced Search Bar (SRCH-01)

**Where:** Top of the existing mail-list pane (replaces the current single-row search area at `inbox.html` lines 101–106).

**Layout:** Two-row header inside the mail-list pane:

```
Row 1: [🔍 Search input___________________________ ] [Zur Inbox] [Neu laden] [✨KI-Frage]
Row 2: [Account chips: Alle | Konto1 | Konto2] [Folder chips (when account selected)] [Datum ▼]
```

**Search input behavior:**
- Debounce: 250ms (existing behavior retained)
- Min characters before search fires: 2 (new — prevents empty FTS5 queries)
- Keyboard: `Enter` executes immediately (bypasses debounce)
- Keyboard: `Escape` clears search and returns to inbox view
- Global shortcut: `/` focuses search input from anywhere on the page
- The existing `searchQuery` x-model and `debounceSearch()` method are extended

**Search results display:**
- Results render in the existing `mails[]` list (same `mail-row` template)
- **New:** Each search result row shows a `match_snippet` line below the subject when `searchActive` is true
- Snippet text uses yellow highlight for `[` `]` delimited FTS5 match markers
- Result count shown in the existing search-active banner: `Suche aktiv: "term" (23 Treffer)`
- "Zur Inbox" button clears search (existing behavior)

**API changes needed:**
- `GET /api/search?q=...&account=...&folder=...&since=...&limit=...`
- Backend must accept `account`, `folder`, `since` parameters and filter the FTS5 JOIN results
- Response includes `match_snippet` field (already returned by `rag_engine.search_emails` but not by `inbox_routes.api_search`)

### 2.2 Account/Folder Filters (SRCH-02)

**Where:** Second row of the mail-list pane header (new filter chip bar).

**Account filter:**
- Renders as a horizontal row of chip-style buttons
- First chip is always "Alle Konten" (equivalent to `isUnified = true`)
- Subsequent chips are populated from the existing `accounts[]` array
- Active chip: filled background (slate-900/slate-100)
- When a specific account is selected, the folder sub-chips appear to the right
- Clicking an account chip filters search results immediately (re-triggers `runSearch()` with `&account=` param)
- The account chip bar is visible both in search mode and inbox mode

**Folder filter:**
- Appears only when a specific account is selected (not "Alle Konten")
- Renders as continuation chips after account chips: "Posteingang", "Gesendet", etc.
- Folder names use the existing `folderDisplayName()` mapping
- Active folder chip: filled background
- Clicking a folder chip adds `&folder=` to the search query

**Date filter:**
- Small dropdown button labeled "Datum" at the end of the filter row
- Options: "Alle", "Letzte 7 Tage", "Letzte 30 Tage", "Letzte 90 Tage", "Dieses Jahr", "Benutzerdefiniert..."
- Selecting a date range adds `&since=` parameter to the search query
- "Benutzerdefiniert..." opens two native date inputs (from/to)

**Combined filter behavior:**
- All filters compose: account + folder + date + query are all AND-combined
- When `searchActive` is false, the filters still work on the regular inbox/folder view
- Clearing search (Zur Inbox button) resets filters to "Alle Konten" state

### 2.3 Attachment/Document Content in Results (SRCH-03)

**Where:** Within the existing FTS5 index and search results.

**Backend requirement (already partially present):**
- The `emails` table `snippet` column already stores email body text in the FTS5 index
- `index.py` FTS5 indexes: `from_addr`, `subject`, `keywords`, `snippet` (which contains body text)
- For SRCH-03, the backend must also index extracted attachment/document text into the `snippet` field or a new `attachment_text` FTS5 column

**UI display:**
- When a search result's `match_snippet` originates from attachment content (not email body), show a small attachment indicator:
  - Lucide icon `paperclip` (12px) with text "aus Anhang" in 11px slate-400
  - This indicator appears inline before the match snippet text
- The `match_snippet` display is the same yellow-highlighted text as body matches
- No separate "search attachments only" filter in Phase 2 — attachment text is searched by default as part of the FTS5 corpus

**API response field addition:**
- `match_source`: `"body"` | `"attachment"` | `"subject"` | `"keywords"` — tells the UI where the match came from
- If backend cannot determine source, default to `"body"`

### 2.4 AI-Assisted Q&A with Grounded Citations (SRCH-04)

**Where:** A slide-in panel/drawer triggered from the search bar area.

**Trigger:**
- A small button with sparkle icon `✨` labeled "KI-Frage" at the right end of the search input row
- Also triggerable via keyboard shortcut: `Shift+/` (i.e. `?`) when the inbox page is focused
- When clicked/triggered, opens the AI Q&A panel as an overlay

**AI Q&A Panel Layout:**

```
┌──────────────────────────────────────────────────────┐
│ ✨ KI-Frage an deine Emails         [Schließen ✕]    │
│                                                      │
│ ┌──────────────────────────────────────────────────┐ │
│ │ Stelle eine Frage zu deinen Emails...            │ │
│ └──────────────────────────────────────────────────┘ │
│                                        [Fragen ▶]    │
│                                                      │
│ Index-Status: 1.234 Emails | llama3.1:8b | ● Online │
│                                                      │
│ ── Letzte Antwort ────────────────────────────────── │
│                                                      │
│   "Welche Emails habe ich letzte Woche von der       │
│    Rettungsdienstschule bekommen?"                    │
│                                                      │
│   Antwort:                                           │
│   Du hast 3 Emails von der Rettungsdienstschule      │
│   in der letzten Woche erhalten:                     │
│   • Am 12.04. einladung zum Fortbildungskurs...      │
│   • Am 10.04. Bestätigung der Anmeldung...           │
│   • Am 08.04. Update zum Prüfungsplan...             │
│                                                      │
│   Quellen:                                           │
│   ┌─ [1] 12.04.2026 | rettungsdienstschule@... │    │
│   │  Einladung Fortbildungskurs "Notfallmedizin" │    │
│   │  [Email öffnen ↗]                              │
│   └──────────────────────────────────────────────────│
│   ┌─ [2] 10.04.2026 | rettungsdienstschule@... │    │
│   │  Bestätigung Anmeldung Frühjahrskurs          │    │
│   │  [Email öffnen ↗]                              │
│   └──────────────────────────────────────────────────│
│                                                      │
│   ── Vorherige Fragen ────────────────────────────── │
│   • "Wann war die letzte Email von Michael?"         │
│   • "Wie viele Spam-Mails diese Woche?"              │
└──────────────────────────────────────────────────────┘
```

**Panel behavior:**
- Opens as a right-side drawer overlaying the detail pane, width: 480px on desktop
- On mobile (<768px), opens as a full-screen modal
- Panel backdrop: semi-transparent `bg-black/30` click-to-close overlay
- Panel slides in from the right with `transform: translateX` + `transition-transform duration-200`
- Content area is scrollable independently

**Q&A input:**
- Text input with placeholder: "Stelle eine Frage zu deinen Emails..."
- Submit button: "Fragen" with right-arrow icon
- Submit via `Enter` key or button click
- Submit sends `POST /api/rag/query` with `{query: input, limit: 10}`

**Loading state:**
- Input disables, submit button shows spinner + "Antwort wird generiert..."
- Animated dots or pulsing sky-600 bar to indicate Ollama processing
- Timeout: if no response after 30 seconds, show error state

**Answer display:**
- User's question shown as a quote block (left border accent, italic)
- AI answer rendered as plain text with markdown-lite formatting (bold, bullet points)
- Answer text at 14px, line-height 1.6 for readability

**Source citations:**
- Each source rendered as a compact card with:
  - Index number `[1]`
  - Date (short format: DD.MM.YYYY)
  - Sender (truncated to 30 chars)
  - Subject (truncated to 50 chars)
  - "Email öffnen" link/button (opens the email in the detail pane)
- Cards are stacked vertically with 8px gap
- Clicking "Email öffnen" (`↗`) closes the AI panel and opens the email in the detail pane
  - This requires looking up the email by `account + folder + msg_uid` from the source data
  - If the email is not in the current `mails[]` list, fetch it via `GET /api/email/{account}/{folder}/{uid}`

**RAG status indicator:**
- Small status line at the top of the panel: "{N} Emails | {model} | ● Online/Offline"
- Fetched from `GET /api/rag/status` on panel open
- "Online" = Ollama reachable, "Offline" = Ollama not reachable
- When offline, the Q&A input is disabled with message: "Ollama ist nicht erreichbar. Stelle sicher, dass Ollama läuft."

**Query history:**
- Below the current answer, show last 5 previous questions as clickable links
- Clicking a previous question re-executes it
- History fetched from `GET /api/rag/status` or from the `rag_queries` table via a new endpoint
- Each history item shows: truncated question text + timestamp

**Reindex action:**
- Small link-button "Index neu aufbauen" at the bottom of the panel
- Triggers `POST /api/rag/reindex`
- Shows confirmation: "Index wird neu aufgebaut. Dies kann einige Minuten dauern."

|----------|--------|---------|

## Alpine.js State Additions

The following properties are added to the existing `inboxApp()` function:

```javascript
// Filter state
searchAccountFilter: '',      // '' = all accounts
searchFolderFilter: '',       // '' = all folders
searchDateFilter: '',         // '' = all time, '7d', '30d', '90d', 'year', 'custom'
searchDateFrom: '',           // custom from date
searchDateTo: '',             // custom to date

// AI Q&A panel state
showAiPanel: false,           // panel visibility
aiQuestion: '',               // current question input
aiAnswer: null,               // {answer, sources, email_count, query}
aiLoading: false,             // loading state
aiError: '',                  // error message
aiStatus: null,               // {total_emails, model, ollama_url, rag_queries}
aiHistory: [],                // recent questions [{query, created_at}]

// Search results enhancement
focusedResultIndex: -1,       // keyboard navigation index
```

## Empty States

| State | Heading | Body |
|-------|---------|------|
| No search results | Keine Treffer | Keine Emails gefunden für "${searchQuery}". Versuche einen anderen Suchbegriff oder andere Filter. |
| No results with active filters | Keine Treffer mit aktuellen Filtern | "${searchQuery}" lieferte keine Ergebnisse in ${account}/${folder}. Passe die Filter an oder suche in allen Ordnern. |
| FTS5 index empty (no emails indexed) | Kein Suchindex vorhanden | Es sind noch keine Emails im Suchindex. Starte den Sortiervorgang oder baue den Index manuell auf. |
| AI panel — no history | Noch keine Fragen gestellt | Stelle deine erste Frage an deine Emails, z.B. "Welche Emails habe ich diese Woche von X bekommen?" |
| AI panel — Ollama offline | Ollama nicht erreichbar | Die KI-Fragefunktion benötigt einen laufenden Ollama-Dienst. Prüfe, ob Ollama gestartet ist und das Modell geladen wurde. |
| AI answer — no emails found | Keine relevanten Emails | Zu dieser Frage konnten keine passenden Emails im Index gefunden werden. Versuche es mit einer anderen Formulierung. |

## Error States

| Error | Display | Recovery |
|-------|---------|----------|
| Search API error (500/network) | Red error banner: "Suche fehlgeschlagen: {error message}" | "Erneut versuchen" button re-runs search |
| FTS5 query syntax error | Amber warning banner: "Suchbegriff ungültig. Verwende einfache Wörter statt Sonderzeichen." | Auto-suggest: strip special chars and re-search |
| RAG query timeout (>30s) | AI panel shows: "Die Antwort dauert zu lange. Ollama reagiert möglicherweise nicht." | "Erneut versuchen" button |
| RAG query error | AI panel shows red box: "Fehler bei der KI-Anfrage: {error}" | "Erneut versuchen" button |
| Reindex failure | Toast notification: "Index-Neuaufbau fehlgeschlagen: {error}" | Manual CLI fallback hint |

## Integration with Existing Inbox

This phase does **not** create a separate search page. All search and RAG functionality integrates into the existing `templates/inbox.html`:

1. **Search bar enhancement:** The existing search input row (lines 101–106) gains filter chips below it and the "KI-Frage" button
2. **Search results:** Use the existing `mails[]` array and `mail-row` template, extended with `match_snippet` rendering
3. **AI Q&A panel:** New `x-data` section appended to the inbox template, drawn as a fixed-position drawer overlay
4. **Folder pane:** Unchanged — account/folder selection from the sidebar continues to work as inbox navigation; the new filter chips in the search bar provide search-scoped filtering
5. **Detail pane:** Unchanged — clicking a search result or an AI source citation opens the email in the existing detail pane

| `templates/inbox.html` | Extend | Add filter chip row, AI panel HTML, match snippet display, new Alpine methods |
